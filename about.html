• Why?

    We want to enable convenient and automatic access to an emerging "Tree of Life" that is currently scattered and inaccessible. Our effort is inspired by Phylomatic and guided by use cases in which researchers need, not the entire tree of life, but a sub-tree tailored to their research

  • What?

    We envision a system of loosely coupled components that can be configured flexibly, and plugged into workflow systems and other applications-- for an example, see Mesquite-o-Tastic. We like to think of "phylotastic" as an adjective (not a noun), like this:

    An application is phylotastic to the extent that, in response to a user query S, it supplies expert phylogenetic knowledge of S, in a form useful for research, in a timely manner.

    where S is a list of identifiers such as { "Mus musculus", "Homo sapiens", "Pan troglogytes" }. Note:

    • User-oriented (responds to user query)
    • Authoritative (based on expert trees)
    • Research-ready (computable tree w/ provenance)
    • Timely (results while you wait)
