• Why?

    The phylotastic project aims to provide cohesive, flexible, automated access to expert knowledge of the emerging "Tree of Life"— knowledge that currently is scattered and inaccessible. In particular, inspired by frequent use of Phylomatic (which provides access to the plant tree of life), we want to make it easy users to get species trees tailored to their particular research problems (e.g., a phylogeny of "model organisms")

  • What?

    We envision a system of loosely coupled components that can be configured flexibly, and can be integrated into existing workflow environments or applications (e.g., see the Mesquite-o-Tastic demo). We like to think of "phylotastic" as an adjective (not a noun), like this:

    An application is phylotastic to the extent that, in response to a user query S, it supplies expert phylogenetic knowledge of S, in a form useful for research, in a timely manner.

    where S is a list of identifiers such as { "Mus musculus", "Homo sapiens", "Pan troglogytes" }. Note:

    • User-oriented - responds to user query
    • Expert knowledge – exposes expert knowledge to non-expert user
    • Research-ready - provides computable tree (not just a pretty picture) with metadata on sources & methods
    • Timely - provides results while you wait
